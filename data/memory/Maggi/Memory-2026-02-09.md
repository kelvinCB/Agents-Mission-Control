# Transformer Architecture & Self-Attention: Technical Explanation

The Transformer architecture, introduced in the seminal paper "Attention Is All You Need" (Vaswani et al., 2017), fundamentally changed how we approach sequence-to-sequence tasks. Unlike Recurrent Neural Networks (RNNs) which process data sequentially, Transformers rely entirely on an attention mechanism to draw global dependencies between input and output. This parallelizable structure allows for significantly faster training and the ability to learn longer-range relationships within data.

## Core Architectural Overview

A standard Transformer follows an Encoder-Decoder structure:

1.  **Input Embedding:** Token inputs are converted into dense vector representations (embeddings).
2.  **Positional Encoding:** Since the architecture has no inherent notion of sequence order (unlike RNNs), positional information is injected into the embeddings.
3.  **Encoder:** A stack of $N$ identical layers. Each layer contains a Multi-Head Self-Attention mechanism followed by a position-wise Feed-Forward Network (FFN).
4.  **Decoder:** Similar to the encoder, but contains an additional layer that performs Multi-Head Cross-Attention over the encoder's output. It also uses Masked Multi-Head Self-Attention to prevent looking ahead at future tokens during training.

## The Mechanism of Self-Attention

At the heart of the Transformer lies the Scaled Dot-Product Attention. Given a set of queries $Q$, keys $K$, and values $V$, the attention mechanism is computed as:

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

*   **Q (Query):** Represents the current word/token looking for information.
*   **K (Key):** The "label" or identifier in the sequence that the query matches against.
*   **V (Value):** The actual content information to be retrieved.

The steps are:
1.  **Dot Product:** Calculate the similarity between each query and every key. High scores indicate strong relevance.
2.  **Scaling:** The dot products are divided by $\sqrt{d_k}$ (the dimension of the keys) to prevent gradient vanishing during softmax for large dimensions.
3.  **Softmax:** The scores are normalized into probabilities (weights) that sum to 1.
4.  **Weighted Sum:** The final output is a weighted sum of the values, where weights are determined by the attention scores.

## Multi-Head Attention

Rather than performing a single attention function with $d_{\text{model}}$ dimensional keys, values, and queries, the Transformer projects them linearly $h$ times (typically 8) with different learned linear projections to $d_q$, $d_k$, and $d_v$ dimensions.

$$
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)W^O
$$

where $\text{head}_i = \text{Attention}(QW_Q^i, KW_K^i, VW_V^i)$.

**Why Multi-Head?**
It allows the model to jointly attend to information from different representation subspaces at different positions. For example, one head might capture syntactic dependencies (Subject $\leftrightarrow$ Verb), while another captures semantic relationships (Cat $\leftrightarrow$ Animal). This enriches the representation with diverse contextual information.

## Positional Encoding

Since the Transformer contains no recurrence or convolution, it must be injected with information about the relative or absolute position of the tokens in the sequence. The paper uses sine and cosine functions of different frequencies:

$$
PE_{(pos, 2i)} = \sin(pos / 10000^{2i/d_{\text{model}}})
$$
$$
PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i/d_{\text{model}}})
$$

These encodings are added to the embedding vectors. The sinusoidal form allows the model to easily learn to attend to relative positions, as linear transformations of the positional encoding correspond to shifts in position.

## Conclusion

The Transformer's power lies in its ability to process all elements of a sequence in parallel while using self-attention to model complex relationships between distant elements. Multi-head attention provides a robust, multi-faceted understanding of context, and positional encoding ensures the model respects the sequential nature of language. This architecture serves as the backbone for modern LLMs like GPT-4, Llama, and Claude.
