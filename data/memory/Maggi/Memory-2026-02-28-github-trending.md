When state transitions between these branches (e.g., form submission succeeds), the entire component tree unmounts and remounts within a new provider context. This forces Framer Motion to re-initialize the animation features bundle on every state change, causing jank and losing internal component state (e.g., scroll position, focus) in children. Recomendación: Hoist LazyMotion to wrap all conditional branches at the top level: return ( <LazyMotion features={domAnimation}> {isEmailSent ? <SuccessView /> : isValidatingSession ? <LoadingView /> : <FormView />} </LazyMotion> ); Alternatively, move LazyMotion to the router/layout level to cover all auth pages with a single provider.
